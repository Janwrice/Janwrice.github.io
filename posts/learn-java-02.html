前言：上节大致过了一遍 Java 的基础知识，这节补齐 Java 的 OOP 部分。

控制访问与包机制

Java 有 4 级访问权限，以及包机制。

包机制

包机制是一套解决类名冲突、封装代码的工具。是类的命名空间加上一套组织管理单元。

包的声明与规则：

  • Java 的源文件，在第一行必须包含声明所属包
  • 包名与目录结构严格绑定

相当于 C++ 的命名空间，但 C++ 的命名空间只按照逻辑结构划分，而 Java 的命名空间必须与目录结构对应。最好按功能分包。

导入

使用其他包下的类时，需要通过 import 导入类的限定名。

// 导入单个指定类
import java.util.ArrayList;
// 导入包下所有类
import java.util.*;
// 静态导入，调用时可以省略类名
import static java.lang.Math.PI;

访问控制修饰符

4 种访问权限级别：

修饰符 同一类内部 同一个包下 不同包的子类 不同包的非子类 C++ 对应说明
public 可访问 可访问 可访问 可访问 与 C++ public 完全一致
protected 可访问 可访问 可访问 不可访问 C++ 无对应；C++ 的 protected 仅子类可访问，同命名空间下不可访问
default（无修饰符） 可访问 可访问 不可访问 不可访问 C++ 无对应，属于包级私有
private 可访问 不可访问 不可访问 不可访问 与 C++ private 完全一致

对于顶级类，只能使用 public 和 default 两种权限，不能使用 private 或 protected。

📌 理解：包相当于文件夹，类相当于文件。

针对 Android 开发

Android 的四大组件，只能声明为 public，因为需要被不同包调度。

Android SDK 有很多隐藏的 default 或者 private 权限的 API，有的可以通过反射调用。

类的继承与重写

Java 通过 extends 关键字实现类的继承。注意，C++ 一个类可以有多个父类，即多重继承。而 Java 一个类只能有一个父类，是单继承的。

// 父类（超类）
class Animal {
    String name;
    void eat() {
        System.out.println("eating");
    }
}

// 子类（派生类），继承Animal
class Dog extends Animal {
    void bark() {
        System.out.println("barking");
    }
}

子类拥有父类所有非 private 的成员变量与方法。父类的 private 成员子类无法直接调用。同时子类不能继承父类的构造方法，只能通过 super 调用，这点与 C++ 一致。

Java 中所有类都默认直接或间接继承自 java.lang.Object 类，它是整个 Java 的根类。所有对象都可以用 Object 类型的引用指向。

super 关键字

super 关键字可用于访问父类成员，代表对父类对象的引用。

调用父类的构造方法

  • super(参数列表) 写在子类构造方法的第一行。如果没有显式调用 super，实际上会自动调用父类的无参构造。
  • super() 与 this()（本类的其他构造方法）不能同时存在。
class Animal {
    Animal(String name) { ... }
}

class Dog extends Animal {
    Dog(String name) {
        super(name); // 调用父类有参构造，必须在第一行
    }
}

调用父类成员

默认可以直接调用，但当子类重写了父类的方法、隐藏了父类的属性时，可以通过 super 来显式地调用父类的实现。

多态

什么是多态？多态就是同一个指令，因为实际执行的对象不同，产生了不同的行为结果。

为什么说"多态的本质是：编译期看父类，运行期看子类"？

编译器只检查调用的指令对不对，而不关心实际执行的对象。编译器下指令，对象执行得到结果。

多态：方法重写

Java 可以重写父类函数。

@Override 注解，即表示这里的函数是重写过的。重写一个函数时，会覆盖父类的原有实现，是 OOP 多态的核心实现。

方法的重写规则：

  • 方法签名必须完全一致：方法名、参数个数、参数类型、参数顺序必须和父类相同
  • 访问权限只能更宽松
  • 返回值类型要兼容
  • 异常只能更严格
  • 父类被 private、static、final 修饰时，无法被重写

多态：父类引用指向子类对象

  • 方法调用：编译期检查父类是否有该方法，而运行期执行子类重写后的实现。
  • 属性调用：成员变量不具备多态性，具体的运行情况看父类。
class Animal {
    String name = "动物";          // 父类成员变量
    public void showName() {
        System.out.println("我是" + name);
    }
}

class Dog extends Animal {
    String name = "狗";            // 子类同名成员变量
    @Override
    public void showName() {
        System.out.println("我是" + name);
    }
}

public class Test {
    public static void main(String[] args) {
        Animal animal = new Dog(); // 父类引用，指向子类对象

        // 1. 访问成员变量（不具备多态性）
        System.out.println(animal.name);  // 输出：动物（不是狗！）

        // 2. 调用方法（具备多态性）
        animal.showName();                // 输出：我是狗（方法重写生效了）
    }
}

Object 类的几个方法

所有类都继承了 Object 类，都支持以下几个方法：

  • toString() —— 返回对象的字符串表示
  • equals(Object obj) —— 判断两个对象是否相等
  • hashCode() —— 返回对象的哈希值
  • getClass() —— 返回对象的运行时类对象，不可被重写，是反射的基础

反射

什么是反射？

反射就是 Java 程序在运行期间，能够动态地操作任意类的内部信息。即在运行状态下，仅凭类名/对象，就可以获取一个类的所有结构信息。

能够在编译时完全不知道类名的情况下，创建对象、调用方法。

针对 Android 开发

Android 的所有 UI 组件都继承自 View：View → TextView → Button/ImageView...

Activity 的生命周期方法，本质都是重写父类的方法，然后由系统框架回调执行。

适配器模式、Base 基类都是 Android 架构中最常用的继承与重写场景。

抽象类与接口

抽象类

包含抽象方法的类必须为抽象类，抽象类可以没有抽象方法。

抽象方法：用 abstract 修饰，只有方法声明、没有方法体的方法。必须由子类重写。

子类继承抽象类后，必须实现所有抽象方法；如果不实现，子类也必须声明为抽象类。

// 抽象类
abstract class Animal {
    String name;

    // 普通方法：公共逻辑直接实现
    void sleep() {
        System.out.println("sleeping");
    }

    // 抽象方法：仅声明，子类必须重写
    abstract void eat();
}

// 子类继承抽象类，必须实现所有抽象方法，否则子类也得声明为抽象类
class Dog extends Animal {
    @Override
    void eat() {
        System.out.println("dog eats food");
    }
}

接口

先前说 Java 不具备多继承，只可以单继承。而接口是 Java 实现"多继承"的核心方式。

接口通过 interface 关键字定义，类通过 implements 关键字实现接口，一个类可以同时实现多个接口。

接口是"行为契约"，只定义"具备什么能力"。

简单理解，就是：可以多继承的抽象类。但存在诸多限制，如接口的成员变量不能重写。实际开发过程中与抽象类混合使用。

// 1. 抽象类：抽取公共代码和状态（省得每个子类重复写）
public abstract class BaseController {
    protected Logger log = LoggerFactory.getLogger(getClass()); // 继承状态
    protected void logRequest() { log.info("收到请求"); }      // 复用逻辑
}

// 2. 接口：定义外部能力契约（方便替换实现，便于测试）
public interface PaymentService {
    void pay(BigDecimal amount);  // 只定义契约
}

// 3. 最终子类：既复用父类代码，又实现多套能力
public class AlipayController extends BaseController implements PaymentService, Serializable {
    @Override
    public void pay(BigDecimal amount) {
        logRequest(); // 用父类的
        // 具体支付宝支付逻辑
    }
}

函数式接口（SAM 接口）

有且仅有一个抽象方法的接口，称为函数式接口。安卓开发中使用非常高频。只要是函数式接口，就可以用 Lambda 表达式简化写法，而无需写内部类的结构。

可以用 @FunctionalInterface 标记。

@FunctionalInterface
interface OnClickListener {
    void onClick();
}

针对 Android 开发

Android 中创建的 View.OnClickListener、Runnable、Callable 都是函数式接口。所以这些回调可以简化成 Lambda。

Android 中所有 UI 点击、滑动、网络请求等全部基于接口实现，匿名内部类 + 接口是 Android 最基础的编程范式。

在一些框架中，View 层与逻辑层通过接口交互，解耦实现与依赖。

内部类与匿名内部类

Java 中存在 4 种内部类，其差别在于定义规则与内存差异。

内部类

定义在另一个类内部的类，统称为内部类。为了实现更强的封装性。内部类可以直接访问外部类的私有成员。

内部类分为 4 种：静态内部类、成员内部类、局部内部类、匿名内部类

静态内部类

用 static 修饰的内部类，只能访问外部类的静态成员与静态方法，不能访问非静态成员。创建实例时不需要依赖外部类对象。不持有外部类的实例引用。

class Outer {
    private static int staticNum = 100;
    private int normalNum = 200;

    // 静态内部类
    static class StaticInner {
        void show() {
            System.out.println(staticNum); // 可以访问外部类静态成员
            // System.out.println(normalNum); // 编译报错，不能访问非静态成员
        }
    }
}

// 创建方式
Outer.StaticInner inner = new Outer.StaticInner();

成员内部类

不加任何修饰，直接定义在类内部的类。

持有外部类所有实例的引用。每个内部成员类对象都绑定一个外部类对象。可以访问外部类的所有成员，包括非静态成员。在其他函数中必须先创建外部类实例，才能创建内部类实例。且在内部类中不能定义静态成员。

class Outer {
    private int num = 100;

    // 成员内部类
    class MemberInner {
        void show() {
            System.out.println(num); // 直接访问外部类私有成员
            System.out.println(Outer.this.num); // 显式调用外部类引用
        }
    }
}

// 创建方式：必须先有外部类对象
Outer outer = new Outer();
Outer.MemberInner inner = outer.new MemberInner();

局部内部类

定义在方法、构造器、或代码块内部的类，作用域只限于当前代码块，可以访问外部类的所有成员。不能使用任何访问修饰符。

匿名内部类

即为没有类名的内部类，必须在创建实例的同时声明它要继承的父类或实现的接口，属于一次性使用的临时类。

用于 Android 中的事件回调。也可以用 Lambda 实现。

事件回调：留一个联系方式给系统，当某件事情发生时，系统主动通过这个接口找到你。

interface OnClickListener {
    void onClick();
}

public class Test {
    public static void main(String[] args) {
        // 匿名内部类实现OnClickListener接口
        OnClickListener listener = new OnClickListener() {
            @Override
            public void onClick() {
                System.out.println("按钮被点击了");
            }
        };
        listener.onClick();
    }
}

⚠ 注意：访问方法内的局部变量时，变量必须为 final。匿名内部类持有外部类引用会导致 Activity 无法释放，引发内存泄漏。
