速通 Kotlin：Java 之后的上手笔记

2026-07-19

前言：速通 Java 后，简单上手一下 Kotlin。

Kotlin 基础语法

Kotlin 跑在 JVM 上，编译成字节码。Kotlin 里没有指针、头文件、运算符重载、析构函数等。

变量

Kotlin 里有两种变量：var 是变量，val 是常量（不可变引用）。

声明方式：

var x = 52
var y: Int = 52

类型推断与显式类型声明两种方式都可以。Kotlin 的哲学中，能用 val 就不要用 var。

函数

一种特殊的返回类型 Unit，相当于 C++ 的 void。

fun greet(name: String): String {
    return "Hello, $name"
}

// 单表达式函数可以简写
fun greet(name: String) = "Hello, $name"

fun log(msg: String): Unit {
    println(msg)
}

控制流

if 表达式：Kotlin 里的 if 类似于 C++ 中的三元条件表达式——它本身有返回值。

val max = if (a > b) a else b

when 表达式：相当于 switch 的增强版。

when (x) {
    1 -> println("one")
    2, 3 -> println("two or three")     // 多个值
    in 4..10 -> println("4 to 10")      // 范围匹配
    is String -> println("got string")  // 类型匹配
    else -> println("something else")   // 默认（类似 default）
}

for 循环：Kotlin 只有 foreach 风格的 for 循环，没有传统的三段式 for(int i = 0; i < n; ++i) 语法。

for (i in 1..5) {           // 包含 5：1, 2, 3, 4, 5
    println(i)
}

for (i in 0 until 5) {      // 不包含 5：0, 1, 2, 3, 4
    println(i)
}

for (i in 5 downTo 1) {     // 递减：5, 4, 3, 2, 1
    println(i)
}

for (i in 0..10 step 2) {   // 步长：0, 2, 4, 6, 8, 10
    println(i)
}

类

Kotlin 的类声明非常简洁，构造参数直接写在类名后面：

class Person(val name: String, var age: Int) {
    fun greet() = "Hi, I'm $name, $age years old"
}

// 这里类的参数相当于构造函数里的参数初始化

构造函数：通过 init 块执行初始化逻辑。

class Person(val name: String, var age: Int) {
    init {
        require(age >= 0) { "Age can't be negative" }
        println("Person created: $name")
    }
}

继承

Kotlin 中类和函数默认都是 final 的，需要显式标记 open 才能被继承或重写。

open class Animal(val name: String) {   // open = 允许被继承
    open fun sound() = "..."            // open = 允许被重写
}

class Dog(name: String) : Animal(name) {
    override fun sound() = "Woof!"
}

📌 与 Java 的区别：Java 中方法默认可以重写（除非标记 final），Kotlin 则反过来——默认不可重写，需主动标记 open。这种设计减少了意外重写带来的问题。

处理可为 Null 的对象

如果一个变量可以为 null，且需要访问它的属性：

fun main() {
    var favoriteActor: String? = "Sandra Oh"
    println(favoriteActor.length)
}

此时编译器会报错。因为 Kotlin 通过语法规则强制实现 null 安全——保证不会意外调用可能为 null 的变量。但这不表示变量不能为 null，只是说当你声明了一个可空类型（String?），就必须显式处理 null 的情况。

使用 ?. 安全调用运算符

fun main() {
    var favoriteActor: String? = "Sandra Oh"
    println(favoriteActor?.length)
}

此时如果 favoriteActor 被赋值为 null，整个表达式会短路并输出 null，而不会抛出异常。

使用 !! 非 null 断言运算符

还可以使用非 null 断言来访问可能为 null 的变量。但此时如果变量真的是 null，程序会抛出 NullPointerException。也就是说，使用非 null 断言时需要自行确保 null 安全，或适当处理异常。

fun main() {
    var favoriteActor: String? = "Sandra Oh"
    println(favoriteActor!!.length)
}

使用 if / else 条件检查

可以用 if 来检查对象是否为 null：

fun main() {
    var favoriteActor: String? = "Sandra Oh"

    if (favoriteActor != null) {
        println("The number of characters in your favorite actor's name is ${favoriteActor.length}.")
    }
}

if / else 表达式：

fun main() {
    var favoriteActor: String? = "Sandra Oh"

    val lengthOfName = if (favoriteActor != null) {
        favoriteActor.length
    } else {
        0
    }

    println("The number of characters in your favorite actor's name is $lengthOfName.")
}

使用 ?: Elvis 运算符

?: 是 Elvis 运算符，可以与 ?. 安全调用运算符搭配使用。如果左侧为 null，则取右侧的默认值：

fun main() {
    var favoriteActor: String? = "Sandra Oh"

    val lengthOfName = favoriteActor?.length ?: 0

    println("The number of characters in your favorite actor's name is $lengthOfName.")
}

这基本等价于上面 if/else 表达式的写法，但简洁很多。实际开发中 ?. + ?: 是最常用的组合。

Lambda 表达式

Lambda 真正的设计理念，实际上是为了实现一个"我写了一段代码，但我不关心它具体在哪里运行，只要把它交给需要这个函数的接口就好"的功能。

所以 Kotlin 这些语言会搞出"尾随 Lambda"这种奇怪的语法，就是深刻体现了这个思想——这个语法明确地表示：给要调用的函数实现一段逻辑。

fun trickOrTreat(isTrick: Boolean, extraTreat: (Int) -> String): () -> Unit {
    if (isTrick) {
        return trick
    } else {
        println(extraTreat(5))
        return treat
    }
}

fun main() {
    val coins: (Int) -> String = { quantity ->
        "$quantity quarters"
    }

    val cupcake: (Int) -> String = {
        "Have a cupcake!"
    }

    val treatFunction = trickOrTreat(false, coins)
    val trickFunction = trickOrTreat(true, cupcake)
    treatFunction()
    trickFunction()
}

Lambda 的简写

Kotlin 提供了很多 Lambda 的「语法糖」，比如尾随 Lambda、以及可以直接把一段代码当变量传给函数，不需要中间变量，实现真正的「匿名函数」：

fun main() {
    val treatFunction = trickOrTreat(false, { "$it quarters" })
    val trickFunction = trickOrTreat(true, null)
    treatFunction()
    trickFunction()
}

当 Lambda 只有一个参数时，可以用隐式名称 it 代替显式参数声明。

repeat 函数

如果一个函数会返回或者接收另一个函数作为实参，该函数就称之为高阶函数。上文的 trickOrTreat 就是一个高阶函数。高阶函数与 Lambda 密不可分。

repeat 是一个常用的高阶函数，它使用函数表达 for 循环的间接方式：

for (i in 1..5) {
    println("第 $i 次")
}

repeat(5) { i ->
    println("第 ${i + 1} 次")  // 注意：i 从 0 开始
}
