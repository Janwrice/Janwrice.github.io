SQL 全面参考指南

2026-07-10

1. SQL 执行顺序（核心心法）

这是最重要的概念。写SQL时脑中必须始终有这个顺序。

书写顺序（你写的）           实际执行顺序（引擎跑的）
─────────────────────      ─────────────────────────
SELECT     ← 6             FROM        ← 1
FROM       ← 1             JOIN        ← 2
JOIN       ← 2             ON          ← 3
ON         ← 3             WHERE       ← 4
WHERE      ← 4             GROUP BY    ← 5
GROUP BY   ← 5             HAVING      ← 6
HAVING     ← 6             SELECT      ← 7  (计算表达式、别名)
ORDER BY   ← 7             DISTINCT    ← 8
LIMIT      ← 8             ORDER BY    ← 9
                           LIMIT       ← 10

窗口函数：在 SELECT 阶段计算（介于 HAVING 之后、ORDER BY 之前）。

关键推论：

  • WHERE 中不能用 SELECT 中定义的别名（WHERE 先执行）
  • ORDER BY 中可以用别名（ORDER BY 后执行）
  • HAVING 中不能用 SELECT 的别名（但MySQL做了扩展允许）
  • 聚合函数 COUNT/SUM/AVG/MAX/MIN 在 GROUP BY 阶段计算

2. 基础查询

2.1 基本SELECT

-- 去重
SELECT DISTINCT column1, column2 FROM table;

-- 别名（AS可省略）
SELECT col AS alias_name FROM table;

-- 限制返回行数
-- MySQL / PostgreSQL: LIMIT
SELECT * FROM table LIMIT 10 OFFSET 20;  -- 跳过20行，取10行

-- SQL Server / Oracle 12c+: FETCH
SELECT * FROM table ORDER BY col OFFSET 20 ROWS FETCH NEXT 10 ROWS ONLY;

2.2 WHERE 过滤

-- 比较:  =  >  <  >=  <=  <>  !=
-- 范围:  BETWEEN ... AND ...   (闭区间)
-- 集合:  IN (val1, val2, ...)
-- 模式:  LIKE 'pattern'       (% 任意多个字符, _ 单个字符)
-- 空值:  IS NULL / IS NOT NULL
-- 逻辑:  AND / OR / NOT       (注意括号控制优先级!)

-- NULL 的注意事项：
-- NULL 参与任何运算结果都是 NULL
-- NULL = NULL  → NULL (不是TRUE!)
-- 需要用 IS NULL 判断

2.3 ORDER BY

-- 多列排序
SELECT * FROM employees ORDER BY department ASC, salary DESC;

-- 用数字引用SELECT列（不推荐）
SELECT name, salary FROM t ORDER BY 2 DESC;  -- 按第2列salary排序

3. 聚合与分组

3.1 聚合函数

函数用途注意
COUNT(*)统计行数（含NULL）最常用
COUNT(col)统计非NULL数量忽略NULL
COUNT(DISTINCT col)统计去重非NULL数量
SUM(col)求和忽略NULL
AVG(col)平均值SUM/COUNT，忽略NULL
MAX(col) / MIN(col)最大/最小值忽略NULL
GROUP_CONCAT(col)MySQL分组拼接GROUP_CONCAT(name ORDER BY name SEPARATOR ',')
STRING_AGG(col, ',')PostgreSQL/标准SQL拼接

3.2 GROUP BY 与 HAVING

-- GROUP BY：将行分组，每组输出一行
-- HAVING：过滤分组（WHERE 过滤原始行，HAVING 过滤聚合后的组）

SELECT department, COUNT(*) AS cnt, AVG(salary) AS avg_sal
FROM employees
WHERE hire_date >= '2020-01-01'      -- 先过滤原始行
GROUP BY department                    -- 再分组
HAVING AVG(salary) > 50000            -- 再过滤分组
ORDER BY cnt DESC;                    -- 最后排序

GROUP BY 重要规则：

  • SELECT 中的非聚合列必须在 GROUP BY 中
  • GROUP BY 可以用列号（不推荐）：GROUP BY 1, 2

4. 多表连接

4.1 JOIN 类型

INNER JOIN     ── 两表都匹配才保留    （交集）
LEFT JOIN      ── 左表全保留，右表无匹配填NULL
RIGHT JOIN     ── 右表全保留，左表无匹配填NULL
FULL JOIN      ── 两表全保留           （并集，MySQL不直接支持）
CROSS JOIN     ── 笛卡尔积            （慎用）
NATURAL JOIN   ── 自动用同名列连接    （慎用，不明确）
SELF JOIN      ── 表和自身连接
-- INNER JOIN
SELECT e.name, d.name AS dept
FROM employees e
INNER JOIN departments d ON e.dept_id = d.id;

-- LEFT JOIN（最常用来查"有没有"）
SELECT e.name, o.order_id
FROM customers c
LEFT JOIN orders o ON c.id = o.customer_id;
-- 如果 o.order_id IS NULL → 该客户没下过单

-- 多表连接
SELECT ...
FROM a
JOIN b ON a.id = b.a_id
JOIN c ON b.id = c.b_id
LEFT JOIN d ON c.id = d.c_id;

4.2 JOIN vs WHERE子查询

-- 查有订单的客户 — 两种写法
-- JOIN:
SELECT DISTINCT c.* FROM customers c JOIN orders o ON c.id = o.cust_id;

-- 子查询（IN）:
SELECT * FROM customers WHERE id IN (SELECT cust_id FROM orders);

-- 子查询（EXISTS）:
SELECT * FROM customers c WHERE EXISTS (SELECT 1 FROM orders o WHERE o.cust_id = c.id);
-- 通常 EXISTS 比 IN 效率好（找到一条就停）

4.3 ON vs USING

-- ON: 任意条件的连接
SELECT * FROM a JOIN b ON a.id = b.a_id AND a.status = 'active';

-- USING: 当两表连接列同名时
SELECT * FROM a JOIN b USING (id);  -- 结果中id列只出现一次

5. 子查询与CTE

5.1 子查询类型

类型位置返回值示例
标量子查询SELECT / WHERE单个值(SELECT MAX(sal) FROM emp)
行子查询WHERE一行WHERE (a,b) = (SELECT ...)
列子查询WHERE IN一列多行WHERE id IN (SELECT ...)
表子查询FROM一张表FROM (SELECT ...) AS t
EXISTSWHERE布尔WHERE EXISTS (SELECT 1 ...)
-- 标量子查询（必须返回一行一列，否则报错）
SELECT name, salary,
       (SELECT AVG(salary) FROM employees) AS company_avg
FROM employees;

-- 相关子查询：子查询引用外层列
SELECT e.name, e.salary
FROM employees e
WHERE e.salary > (SELECT AVG(salary) FROM employees WHERE dept_id = e.dept_id);

-- EXISTS / NOT EXISTS
SELECT * FROM customers c
WHERE EXISTS (SELECT 1 FROM orders o WHERE o.cust_id = c.id AND o.amount > 1000);

SELECT * FROM customers c
WHERE NOT EXISTS (SELECT 1 FROM orders o WHERE o.cust_id = c.id);

5.2 CTE（公用表表达式）

-- 基本CTE：让复杂查询可读、可复用
WITH dept_stats AS (
    SELECT dept_id, AVG(salary) AS avg_sal, COUNT(*) AS cnt
    FROM employees
    GROUP BY dept_id
),
high_paid AS (
    SELECT * FROM employees WHERE salary > 100000
)
SELECT e.name, e.salary, d.avg_sal
FROM high_paid e
JOIN dept_stats d ON e.dept_id = d.dept_id
WHERE e.salary > d.avg_sal;

-- 递归CTE（生成序列 / 树形结构）
WITH RECURSIVE seq(n) AS (
    SELECT 1
    UNION ALL
    SELECT n + 1 FROM seq WHERE n < 10
)
SELECT * FROM seq;  -- 1 到 10

-- 树形结构（组织架构）
WITH RECURSIVE org_tree AS (
    SELECT id, name, manager_id, 1 AS level
    FROM employees WHERE manager_id IS NULL
    UNION ALL
    SELECT e.id, e.name, e.manager_id, t.level + 1
    FROM employees e
    JOIN org_tree t ON e.manager_id = t.id
)
SELECT * FROM org_tree ORDER BY level, id;

何时用CTE vs 子查询：

  • 同一结果被多次引用 → CTE
  • 需要递归 → CTE
  • 让复杂查询分步骤、可读 → CTE
  • 简单一次性过滤 → 子查询即可

6. 窗口函数

窗口函数是SQL高级查询的核心，几乎所有"排名/累计/移动平均/环比"类问题都靠它。

6.1 基本语法

SELECT 
    column,
    窗口函数() OVER (
        PARTITION BY col1, col2   -- 分组（可选）
        ORDER BY col3             -- 排序（部分函数必需）
        ROWS/RANGE BETWEEN ...    -- 窗口帧（可选）
    ) AS alias
FROM table;

6.2 排名函数

-- ROW_NUMBER(): 唯一序号 (1,2,3,4...)
-- RANK():       跳跃排名 (1,1,3,4...)
-- DENSE_RANK(): 连续排名 (1,1,2,3...)
-- NTILE(n):     分成n组，返回组号(1到n)

SELECT name, salary,
    ROW_NUMBER() OVER (ORDER BY salary DESC) AS row_num,
    RANK()       OVER (ORDER BY salary DESC) AS rank_num,
    DENSE_RANK() OVER (ORDER BY salary DESC) AS dense_num,
    NTILE(4)     OVER (ORDER BY salary DESC) AS quartile
FROM employees;

-- 经典题：查每个部门工资第2高的员工
WITH ranked AS (
    SELECT *, DENSE_RANK() OVER (PARTITION BY dept_id ORDER BY salary DESC) AS rnk
    FROM employees
)
SELECT * FROM ranked WHERE rnk = 2;

6.3 偏移函数（LAG / LEAD）

-- LAG(col, offset, default):  往上取（看前面）
-- LEAD(col, offset, default): 往下取（看后面）

-- 每个员工和前一个入职的人比工资
SELECT name, salary, hire_date,
    LAG(salary) OVER (ORDER BY hire_date) AS prev_salary,
    salary - LAG(salary) OVER (ORDER BY hire_date) AS diff
FROM employees;

-- 每天销售额和前一天比较（环比）
SELECT sale_date, amount,
    LAG(amount) OVER (ORDER BY sale_date) AS prev_day_amount,
    amount - LAG(amount) OVER (ORDER BY sale_date) AS change
FROM daily_sales;

6.4 聚合窗口函数

-- SUM / AVG / COUNT / MAX / MIN 都可以做窗口函数
-- 加 ORDER BY → 累计效果

-- 累计求和
SELECT sale_date, amount,
    SUM(amount) OVER (ORDER BY sale_date) AS cumulative
FROM daily_sales;

-- 移动平均（前2行+当前行+后2行）
SELECT sale_date, amount,
    AVG(amount) OVER (ORDER BY sale_date ROWS BETWEEN 2 PRECEDING AND 2 FOLLOWING) AS ma5
FROM daily_sales;

-- 每个部门内工资累计占比
SELECT dept_id, name, salary,
    SUM(salary) OVER (PARTITION BY dept_id ORDER BY salary) * 1.0 
        / SUM(salary) OVER (PARTITION BY dept_id) AS cum_ratio
FROM employees;

6.5 首末值函数

-- FIRST_VALUE(col): 窗口内第一个值
-- LAST_VALUE(col):  窗口内最后一个值（注意帧范围！）
-- NTH_VALUE(col, n): 窗口内第n个值

SELECT dept_id, name, salary,
    FIRST_VALUE(name) OVER (PARTITION BY dept_id ORDER BY salary DESC) AS top_earner
FROM employees;

6.6 窗口帧

-- ROWS: 物理行为单位
-- RANGE: 逻辑值为单位（ORDER BY列值相同视为同行）

-- 常用帧：
ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW   -- 从第一行到当前行
ROWS BETWEEN 2 PRECEDING AND CURRENT ROW            -- 前2行+当前行
ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING            -- 前1+当前+后1
ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING  -- 整个分区

7. 条件与分支

7.1 CASE WHEN

-- 两种写法：
-- 搜索CASE（推荐，灵活）
CASE WHEN condition1 THEN result1
     WHEN condition2 THEN result2
     ELSE default_result
END

-- 简单CASE
CASE column WHEN value1 THEN result1
            WHEN value2 THEN result2
            ELSE default_result
END

-- 示例：成绩分级
SELECT name, score,
    CASE WHEN score >= 90 THEN 'A'
         WHEN score >= 80 THEN 'B'
         WHEN score >= 70 THEN 'C'
         WHEN score >= 60 THEN 'D'
         ELSE 'F'
    END AS grade
FROM students;

-- CASE 在聚合中做条件统计（交叉表/透视）
SELECT 
    SUM(CASE WHEN status = 'completed' THEN 1 ELSE 0 END) AS completed_cnt,
    SUM(CASE WHEN status = 'pending' THEN 1 ELSE 0 END) AS pending_cnt,
    COUNT(*) AS total
FROM orders;

7.2 COALESCE / NULLIF / IFNULL

-- COALESCE(val1, val2, ...) : 返回第一个非NULL值（标准SQL）
SELECT name, COALESCE(phone, '无电话') AS phone FROM contacts;

-- NULLIF(a, b): 如果 a=b 返回NULL，否则返回a（避免除零）
SELECT amount / NULLIF(count, 0) AS avg_price FROM sales;

-- IFNULL(a, b): MySQL特有，等同 COALESCE(a, b)
-- NVL(a, b): Oracle中
-- IF(score >= 60, '及格', '不及格'): MySQL特有（条件判断）

9. 集合运算与高级操作

-- UNION: 合并去重
-- UNION ALL: 合并保留所有行（推荐）
-- INTERSECT: 交集（MySQL 8.0不支持）
-- EXCEPT: 差集（MySQL 8.0不支持）

-- MySQL实现差集：用 LEFT JOIN ... IS NULL

-- MySQL实现交集：用 INNER JOIN 或 IN
SELECT DISTINCT a.id FROM a INNER JOIN b ON a.id = b.id;

13. 解题思考模型

面对一道SQL题目，按以下五步走。

步骤1：识别数据实体和关系

  • 题目涉及几张表？分别是干什么的？
  • 表之间怎么关联？（外键是谁？）
  • 主键是什么？（每行是否唯一？）

步骤2：翻译需求为"数据操作"

需求关键词SQL操作
"每个"/"按XX分类"GROUP BY XX
"排名"/"前N名"/"第N高"窗口函数 ROW_NUMBER/RANK
"是否存在"/"有没有"EXISTS / NOT EXISTS / LEFT JOIN ... IS NULL
"和前一天/上一期比"LAG / LEAD
"累计"SUM() OVER (ORDER BY ...)
"占比"col / SUM() OVER (PARTITION BY ...)
"连续N天/次"ROW_NUMBER + 日期差 巧妙构造
"同时包含A和B"HAVING 中做条件计数
"只包含A不包含B"LEFT JOIN ... IS NULL 或 NOT EXISTS
"条件计数"/"透视表"CASE WHEN + SUM
"每组取TOP N"ROW_NUMBER + 子查询取 rnk<=N

步骤3-5：骨架→粒度→打磨

  • 先搭 FROM + JOIN 骨架
  • 判断是否需要 GROUP BY（每组一行 vs 保留明细）
  • 按执行顺序过一遍：WHERE → GROUP BY → HAVING → SELECT → ORDER BY

核心解题套路

套路A：每组取Top N

WITH ranked AS (
    SELECT *, ROW_NUMBER() OVER (PARTITION BY group_col ORDER BY value_col DESC) AS rn
    FROM table
)
SELECT * FROM ranked WHERE rn <= N;

套路B：连续出现/连续天数

WITH numbered AS (
    SELECT *, ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY date) AS rn
    FROM logins
),
grouped AS (
    SELECT *, DATE_SUB(date, INTERVAL rn DAY) AS grp
    FROM numbered
)
SELECT user_id, MIN(date), MAX(date), COUNT(*) AS consecutive_days
FROM grouped
GROUP BY user_id, grp
HAVING COUNT(*) >= N;

套路C：留存率/转化率

WITH first_day AS (
    SELECT DISTINCT user_id, date FROM activity
)
SELECT COUNT(DISTINCT r.user_id) * 1.0 / COUNT(DISTINCT f.user_id) AS retention
FROM first_day f
LEFT JOIN first_day r ON f.user_id = r.user_id 
    AND r.date = DATE_ADD(f.date, INTERVAL 1 DAY);

套路D：同时满足多个条件

SELECT student_id FROM enrollments
WHERE course_id IN ('A', 'B')
GROUP BY student_id
HAVING COUNT(DISTINCT course_id) = 2;

套路E：中位数

WITH ranked AS (
    SELECT *, ROW_NUMBER() OVER (ORDER BY value) AS rn,
              COUNT(*) OVER () AS total
    FROM table
)
SELECT AVG(value) FROM ranked
WHERE rn BETWEEN total/2 AND total/2 + 1;

套路F：环比/同比

SELECT month, amount,
    LAG(amount) OVER (ORDER BY month) AS prev_month,
    (amount - LAG(amount) OVER (ORDER BY month)) * 1.0 
        / NULLIF(LAG(amount) OVER (ORDER BY month), 0) AS mom_change
FROM monthly_sales;

DEBUG 思考清单

  1. JOIN类型对吗？ 该 LEFT JOIN 的用了 INNER JOIN？
  2. WHERE导致LEFT JOIN变INNER JOIN了吗？ 对右表的条件放ON里别放WHERE里！
  3. NULL值处理了吗？
  4. GROUP BY列够吗？
  5. WHERE和HAVING用反了吗？
  6. 重复行导致COUNT不对吗？ 该用 COUNT(DISTINCT) 还是 COUNT(*)？
  7. 整数除法了吗？ 3/2=1 → 乘 1.0
  8. LIMIT没加ORDER BY？ 结果随机！

最后一条心法：SQL不是"怎么写"，而是"我想要什么结果集"。
