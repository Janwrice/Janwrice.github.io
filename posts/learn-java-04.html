前言：最后熟悉一下 Java 的高级用法。

Lambda 表达式

Java 的 Lambda 必须依赖于函数式接口。Java 实际上没有 Lambda，只是函数式接口的一种语法糖。

// 1. 完整匿名内部类写法
button.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View v) {
        Toast.makeText(MainActivity.this, "点击了", Toast.LENGTH_SHORT).show();
    }
});

// 2. Lambda简化写法
button.setOnClickListener(v -> Toast.makeText(MainActivity.this, "点击了", Toast.LENGTH_SHORT).show());

语法简化

// 完整写法
(String a, String b) -> {
    return a.length() - b.length();
};

// 最简写法
(a, b) -> a.length() - b.length();

Java 多线程基础

Java 的线程有两种创建方式。提供生命周期状态转换、synchronized 内置锁的同步机制。

Java 线程采用 1:1 内核线程模型，每个 Java 线程直接映射成操作系统的原生内核线程，由操作系统统一调度。

多线程的创建

方式一：继承 Thread 类

class MyThread extends Thread {
    @Override
    public void run() {
        // 线程执行的任务代码
        System.out.println("子线程运行中: " + getName());
    }
}

// 启动线程
public class Test {
    public static void main(String[] args) {
        MyThread t = new MyThread();
        t.start(); // 启动新线程，不能直接调用run()
    }
}

方式二：实现 Runnable 接口

class MyTask implements Runnable {
    @Override
    public void run() {
        System.out.println("子线程运行中");
    }
}

// 启动线程
public class Test {
    public static void main(String[] args) {
        Thread t = new Thread(new MyTask());
        t.start();
    }
}

📌 推荐：优先使用 Runnable，Java 单继承限制，实现接口更灵活。仅当需要重写 Thread 的其他方法时，才考虑继承 Thread。

线程的生命周期

状态 含义 触发条件
NEW 新建 线程对象已创建，但尚未调用 start() new Thread() 之后
RUNNABLE 可运行 线程已启动，正在执行或等待 CPU 调度 调用 start() 后，包含就绪、运行两个子状态，由 OS 切换
BLOCKED 阻塞 线程等待获取 synchronized 锁，被阻塞在锁外 进入同步代码块 / 方法时，锁被其他线程持有
WAITING 无限等待 线程主动等待，需被其他线程唤醒 调用 wait()、join()、LockSupport.park() 且无超时时间
TIMED_WAITING 计时等待 等待指定时间后自动唤醒 调用 sleep()、wait(time)、join(time)
TERMINATED 终止 线程执行完毕，或因异常退出 run() 方法执行结束、未捕获异常终止

线程同步

多线程同时操作共享变量时，会产生竞态条件，导致数据不一致。Java 最基础的同步手段是 synchronized 关键字，基于每个对象自带的内置监视器锁实现。

常用线程方法

方法 作用 关键特性
Thread.sleep(long ms) 当前线程休眠指定毫秒，进入计时等待 不会释放锁，只是让出 CPU，时间到自动恢复
Thread.yield() 主动让出 CPU，回到就绪状态 不释放锁，仅给调度器提示，不保证立即让出
t.join() 当前线程等待 t 线程执行完毕再继续 可设置超时时间，常用于等待子线程任务完成
wait() / notify() / notifyAll() Object 类方法，用于线程间通信 必须在同步代码块中调用；wait 会释放锁，notify 唤醒一个等待线程
setDaemon(boolean) 设置为守护线程 所有非守护线程结束后，JVM 自动退出，守护线程强制终止

针对 Android 开发

  • 主线程（UI 线程）模型：应用启动时系统创建唯一主线程，负责 UI 绘制、事件分发，是唯一可以直接更新 UI 的线程。本质是一个运行消息循环的 Thread，不断从消息队列取消息执行。
  • 当主线程被阻塞，超过 5 秒就会触发 ANR（应用未响应）。
  • 子线程不能直接操作 UI，必须通过 Handler、runOnUiThread 等机制切回主线程。

注解

Java 内置注解

注解 作用目标 说明
@Override 方法 编译期检查方法是否正确重写父类 / 接口方法，签名不匹配时报错
@Deprecated 类、方法、字段 标记元素已废弃，编译器给出使用警告
@SuppressWarnings 类、方法、字段 抑制指定类型的编译警告，如 "unchecked"、"deprecation"、"rawtypes"、"all"
@FunctionalInterface 接口 标记函数式接口（仅一个抽象方法），编译期校验
@SafeVarargs 方法、构造器 抑制泛型可变参数的堆污染警告，仅用于 final / static / private 构造器

Java 元注解

注解 说明 常用取值
@Target 限制注解可作用的元素类型 ElementType.FIELD、METHOD、TYPE、PARAMETER、CONSTRUCTOR 等
@Retention 指定注解保留周期 RetentionPolicy.SOURCE 源码期、CLASS 编译期、RUNTIME 运行期
@Documented 标记注解将被包含在 Javadoc 文档中 —
@Inherited 允许子类继承父类上的该注解 —
@Repeatable 允许同一位置重复使用该注解 —
