Android 15 状态栏处理：Edge-to-Edge 实践指南

2026-07-18

📌 术语约定：本文中，「下层 / 上层」表示两个组件「覆盖」在一起时的层次效果；「下面 / 上面」表示两个组件在屏幕上的上下排列。

在 Android 15 上，谷歌官方强制让应用启用了 Edge-to-Edge 的沉浸式设计。也就是说，在 Android 15 上，应用的状态栏不再具有淡灰色的背景了，而是采用完全透明的设计，让图标直接显示在界面上层。

而且更进一步地，状态栏无法直接感知到 Compose 可组合项的颜色，也就无法自动对应地更改状态栏图标的颜色。

由此，本文讨论一下 Android 15 上状态栏的几种常用处理方式。

保留状态栏

在 Android 15 上，默认所有情况下——日间模式下状态栏图标色就是黑色的，夜间模式下就是白色的。而且如果不加以配置 Modifier，默认所有的组件都是顶头显示的（即显示在状态栏的下层）。

方式一：使用 innerPadding 避开状态栏

如果想要让界面的 Compose 元素显示在状态栏的下面（即上下排列，不被状态栏覆盖），就需要让界面的 Modifier 接收一个 innerPadding 参数。

class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge()
        setContent {
            HappyBirthdayTheme {
                Scaffold(
                    containerColor = MaterialTheme.colorScheme.background,
                    modifier = Modifier.fillMaxSize()
                ) { innerPadding ->
                    View_1(
                        painterResource(R.drawable.bg_compose_background),
                        title = stringResource(R.string.jetpack_compose_tutorial_title),
                        content_01 = stringResource(R.string.jetpack_compose_tutorial_content_01),
                        content_02 = stringResource(R.string.jetpack_compose_tutorial_content_02),
                        Modifier.padding(innerPadding)
                    )
                }
            }
        }
    }
}

如上代码的 View_1 是一个很简单的组件。使用以上的代码，将 View_1 的 Modifier 的 padding 设置为 Scaffold 提供的 innerPadding，View_1 就正常显示在状态栏下面了。

方式二：不加 innerPadding + 手动设置图标颜色

而如果不接受这个 innerPadding 参数，我们的组件自然就显示在状态栏的下层了（即被状态栏覆盖）：

class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        // 设置状态栏的图标色为白色
        enableEdgeToEdge(
            statusBarStyle = SystemBarStyle.dark(
                scrim = android.graphics.Color.TRANSPARENT
            )
        )
        setContent {
           HappyBirthdayTheme {
               Scaffold(
                   containerColor = MaterialTheme.colorScheme.background,
                   modifier = Modifier.fillMaxSize()
               ) { _ ->
                   View_1(
                       painterResource(R.drawable.bg_compose_background),
                       title = stringResource(R.string.jetpack_compose_tutorial_title),
                       content_01 = stringResource(R.string.jetpack_compose_tutorial_content_01),
                       content_02 = stringResource(R.string.jetpack_compose_tutorial_content_02)
                   )
               }
           }
       }
    }
}

但此时如果背景是深色的，需要手动定义状态栏图标的颜色（如上通过 SystemBarStyle.dark() 将图标设为白色）。

方式三：手动绘制半透明状态栏背景

可以看到即使手动设置了图标颜色，深色背景上直接叠加白色图标，显示效果也不是很好。能不能直接给状态栏上一个半透明的背景呢？也是可以的。

谷歌在 Android 15 移除了状态栏原生背景的设计，但通过下面这个接口，把状态栏的高度信息完全暴露出来了：

Modifier.windowInsetsTopHeight(WindowInsets.statusBars)

因此可以通过它，在我们的界面上层画一个"假状态栏"背景：

class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge(
            statusBarStyle = SystemBarStyle.dark(
                scrim = android.graphics.Color.TRANSPARENT
            )
        )
        setContent {
            HappyBirthdayTheme {
                Scaffold(
                    containerColor = MaterialTheme.colorScheme.background,
                    modifier = Modifier.fillMaxSize()
                ) { _ ->
                    Box(modifier = Modifier.fillMaxSize()) {
                        View_1(
                            painterResource(R.drawable.bg_compose_background),
                            title = stringResource(R.string.jetpack_compose_tutorial_title),
                            content_01 = stringResource(R.string.jetpack_compose_tutorial_content_01),
                            content_02 = stringResource(R.string.jetpack_compose_tutorial_content_02)
                        )
                        // 顶层：画一个深色半透明的"假状态栏"
                        Spacer(
                            modifier = Modifier
                                .fillMaxWidth()
                                // 神奇的 API：自动获取当前手机状态栏的高度
                                .windowInsetsTopHeight(WindowInsets.statusBars)
                                // 设置为你想要的颜色和透明度（这里是 30% 不透明度的黑色）
                                .background(Color.Black.copy(alpha = 0.3f))
                        )
                    }
                }
            }
        }
    }
}

核心思路：用一个 Box 包裹，底层放内容，上层放一个高度等于状态栏、颜色半透明的 Spacer。这样就模拟出了旧版 Android 中状态栏背景的效果，同时图标保持清晰可辨。

不保留状态栏

如果认为状态栏的存在很不方便，可以使用以下方式直接隐藏状态栏。用户需要时向下滑动屏幕即可召唤出状态栏：

class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge(
            statusBarStyle = SystemBarStyle.dark(
                scrim = android.graphics.Color.TRANSPARENT
            )
        )

        // 关键代码：隐藏状态栏
        val controller = WindowCompat.getInsetsController(window, window.decorView)

        controller.hide(WindowInsetsCompat.Type.statusBars())

        controller.systemBarsBehavior =
            WindowInsetsControllerCompat.BEHAVIOR_SHOW_TRANSIENT_BARS_BY_SWIPE

        setContent {
            HappyBirthdayTheme {
                Scaffold(
                    containerColor = MaterialTheme.colorScheme.background,
                    modifier = Modifier.fillMaxSize()
                ) { _ ->
                    View_1(
                        painterResource(R.drawable.bg_compose_background),
                        title = stringResource(R.string.jetpack_compose_tutorial_title),
                        content_01 = stringResource(R.string.jetpack_compose_tutorial_content_01),
                        content_02 = stringResource(R.string.jetpack_compose_tutorial_content_02)
                    )
                }
            }
        }
    }
}

关键点：

  • controller.hide() 直接隐藏状态栏，界面获得完整的屏幕空间
  • BEHAVIOR_SHOW_TRANSIENT_BARS_BY_SWIPE 允许用户通过向下滑动手势临时唤出状态栏，几秒后自动再次隐藏
