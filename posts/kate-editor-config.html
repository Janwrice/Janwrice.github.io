KDE Kate 编辑器配置教程：从文本编辑器到 IDE

2024-09

相信使用 KDE 桌面的朋友对 Kate 编辑器一定不陌生。但你真的会使用 Kate 编辑器吗？

Kate 是一个支持 Windows、Linux、macOS 的功能强大的文本编辑器，并可以通过配置 LSP 服务器来实现自动补全与语法高亮。在 Linux 上，通过简单的配置它就可以作为一款 IDE 来使用。

Kate 由 KDE 团队开发支持，是 GNU 自由软件。在 KDE 桌面上它是自带的。

  • 官方网站：Kate - 多功能的文本编辑利器
  • Windows 版本：可以直接在 Microsoft Store 上下载

在其他 Debian 系的 Linux 发行版上，我们可以使用以下命令安装：

sudo apt install kate

或者可以参考 Kate 的官方网站。对于简单的文本修改，它开箱即用，并不需要做太多的配置。

鉴于这是个 Linux 论坛（迫真），我们详细介绍一下它在 Linux 上的配置。

Linux 下配置自动补全与自动编译运行

配置 C++ 自动补全

Kate 编辑器配置自动补全很容易，安装一个 clangd 的 LSP 服务器即可：

sudo apt install clangd

安装完后我们打开 Kate，打开一个 C++ 代码文件。此时不出意外 Kate 会自动检测到 Clangd 的存在并自动配置 LSP 服务器。也就是 Kate 现在已经可以自动补全代码了，如图：

我们选择菜单栏的 工具 → 外部工具 → 编译运行 即可完成编译运行，默认会输出在下面的外部工具栏中，比如这个 Hello world 程序：

如果没有看到输出，可能是你没有安装 g++，apt 安装一个即可：

sudo apt install g++

不出意外的话你可以在下面的外部工具输出栏看到输出了。但这个输出栏并不能交互。对于稍微复杂一点的程序这个输出就不能使用了。

配置终端输出

我们可以选择菜单栏的 工具 → 外部工具 → 配置 中找到 编译运行 C++ 这一项选择编辑：

在 KDE 桌面环境下，可以将 参数 一栏更改为：

-c 'g++ "%{Document:FilePath}" -o "%{Document:FileBaseName}"  && konsole --hold --noclose -e "./%{Document:FileBaseName}"'

当然如果你使用的是其他桌面，请将 konsole 一词更改为你自己使用的终端。比如你使用的是 GNOME，那么应该改成 gnome-terminal。

这样再次使用此外部工具时，就可以在我们的终端中运行我们的程序了：

设置快捷键

我们还可以为此工具增加一个快捷键。我们选择菜单栏的 配置 → 配置键盘快捷键，在里面找到 编译运行 C++，就可以自定义快捷键了。比如我这里设置为 Alt+R：

这样，我们直接按 Alt+R 就可以一键编译运行我们的代码了。

至此，Kate 拥有了 C++ 的代码补全与自动编译运行的功能。

配置 Python 开发环境

其他的语言可以以此为参考。比如我们要运行 Python 代码：

我们可以看到它提示我们没有找到 pylsp，我们去安装一个 pylsp 即可：

sudo apt-get install python3-pylsp

这样我们就可以在 Python 代码中使用自动补全了。

要自动编译 Python 代码，在外部工具中选择 配置 → 添加工具：

  • 可执行文件选择 sh
  • 参数填：
-c 'konsole --hold --noclose -e "python3 %{Document:FilePath}"'

保存即可。这样我们就可以在外部工具里运行 Python 代码了。

总结

其他的语言都可以以此作为参考。Kate 支持多种语言，个人认为配置好是要比 VS Code 好用的。

提示：如果你使用的是非 KDE 桌面环境，记得将 konsole 替换为你自己使用的终端模拟器（如 gnome-terminal、xfce4-terminal 等）。
