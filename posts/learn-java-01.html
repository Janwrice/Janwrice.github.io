一位 C++ 程序员的安卓开发 Java 速通 0x01

2026-07-12

前言：我已经学会了 C++，不过最近想要上手 Android 开发，决定速通一下 Java 和 Kotlin。

Java 速通 —— Java 基础

Java 运行机制

什么是对象？类的一个实例即为一个对象。

Java 的运行环境

分为三层，JDK、JRE、JVM：

  • JVM：Java 的虚拟机，用于执行 Java 编译后的字节码文件
  • JRE：JVM + Java 的核心类库，要运行 Java 程序，必须要有 JRE
  • JDK：Java 的开发环境

Java 的运行流程

Java 需要先经过编译期，通过 javac 将 .java 编译成 .class 字节码文件，相当于 Java 的机器码后，再运行。

运行时，通过 java 命令启动 JVM，执行 .class 文件。采用解释 + JIT 即时编译技术。因为全靠 JVM 解释，所以可以 一次编译，到处运行。

程序入口

Java 是一门完全面向对象的语言，不存在全局函数，一次程序入口必须写在类中，由 JVM 负责执行。

public class Main {
    public static void main(String[] args) {
        System.out.println("hello, world");
    }
}
  • public：表示可以从外部调用
  • static：表示静态成员函数，即 JVM 调用入口时不需要创建类的实例，可以直接调用
  • void：Java 的 main 函数没有返回值
  • main：Java 的 main 函数标识符

⚠ 注意：main 方法中的 String[] args 不能省略，这是 Java 的语法规范严格定义的。

针对 Android 开发

在 Android 中应用没有 main 方法，是因为 Android 重新封装了 JDK，"四大组件"的生命周期完全由系统负责。

Android 安装包中的 .dex 文件，本质就是 class 转换来的 Android 平台的专属字节码。

四大组件

四大组件是 Android 应用的核心功能单元，它们由 Android 系统统一管理生命周期。本质都是 Java 的类。

  • Activity 活动
    • 负责 UI 界面、处理用户交互，即一个页面
  • Service 服务
    • 后台运行的长期任务，无界面，退出前台界面仍然可以继续运行，如音乐播放、下载服务等
  • BroadcastReceiver 广播接收器
    • 订阅并接收事件，触发对应逻辑
  • ContentProvider 内容提供者
    • 用于共享数据，封装数据读写

📌 提示：除广播外，其他组件都必须在 AndroidManifest.xml 文件中注册后才可以使用。

数据类型与参数传递

数据类型

Java 中的数据类型分为基本数据类型和引用数据类型两类。

8 种基本数据类型

类型 含义 取值
byte 字节型整数 -128 ~ 127
short 短整型 -32768 ~ 32767
int 整型 $-2^{31}$ ~ $2^{31}-1$
long 长整型 $-2^{63}$ ~ $2^{63}-1$
float 单精度浮点 $\pm 3.4e38$
double 双精度浮点 $\pm 1.7e308$
char 字符型 Unicode 字符集
boolean 布尔型 true / false

这 8 种基本类型不是对象，有时候需要把他们转换成对象。

包装类与自动装箱 / 拆箱

// 自动装箱：编译器自动转为 Integer.valueOf(100)
Integer num = 100;
// 自动拆箱：编译器自动转为 num.intValue()
int n = num;

引用数据类型

所有非基本类型都是引用类型，包括类、接口、数组、枚举等。

  • 引用变量本身存储在栈内存中，保存的是对象在堆内存中的地址。
  • 对象本身永远存在堆内存中。

参数传递

🔑 核心结论：在 Java 中只有值传递，所谓的传递的引用，只是地址的一个副本，而不是真正的引用传递。

基本类型传参

public static void change(int a) {
    a = 200;
}

public static void main(String[] args) {
    int num = 100;
    change(num);
    System.out.println(num); // 此时 num 仍然为 100
}

引用类型传参

public static void changeArray(int[] arr) {
    arr[0] = 999;
    arr = new int[]{1, 2, 3};
}

public static void main(String[] args) {
    int[] array = {10, 20, 30};
    changeArray(array);
    System.out.println(array[0]); // 999
}

📌 关键理解：Java 可以修改堆中的对象，但不能让一个引用指向另一个对象。如上的例子中，array[0] 变成了 999，但整个对象并不会变成后边定义的 {1, 2, 3}。

针对 Android 开发

Android 中的集合、泛型是对象，只能使用包装类。其中会频繁地自动拆装箱，可能会产生性能损耗。

数组与字符串

数组

Java 中的数组是定长的引用类型对象，一旦初始化后，长度不可修改，与 C++ 的 vector 不同。

数组的声明

int[] arr1;

数组的初始化

// 列表初始化
int[] arr = {1, 2, 3, 4, 5};

// 先声明后赋值
int[] arr1;
arr1 = new int[]{1, 2, 3, 4, 5};

动态初始化

// 声明时指定长度
int[] arr = new int[5];

二维数组（数组的数组）

int[][] matrix = new int[3][4];

int[][] irregular = new int[3][];
irregular[0] = new int[2];
irregular[1] = new int[4];

⚠ 与 C++ 的区别：C++ 原生的多维数组是连续的内存，每行长度必须一致；而 Java 中每行独立，内存不连续。

字符串

String 是 Java 中最常用的引用类型，核心特性是不可变。其对象一旦创建，内部的字符就永远无法修改，是常量。需要修改只能创建一个新的对象。其设计理念是为了实现字符串常量复用。

字符串比较

🔑 核心结论：Java 里 == 比较的是两个引用的内存地址，即是否指向的是同一个对象。所以直接使用 == 比较两个字符串是否相等是不可行的。需要使用字符串的 .equals() 方法。
